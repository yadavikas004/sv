Chapter 1 — Introducing VMware vSphere 6.7

Products, Features, Licensing, Why vSphere?

What is VMware vSphere 6.7?
🔴 Must Know
Definition: VMware vSphere 6.7 is a server virtualization platform (suite) that enables organizations to consolidate workloads, optimize resources, and simplify IT management. Its core is the VMware ESXi hypervisor.
  • Provides a foundation for virtual machines and management tools
  • Designed for high performance, reliable management, and scalability
  • Integrates with modern hardware efficiently
  • Central suite for server virtualization across enterprise IT
💡

Chapter 2 — Planning & Installing VMware ESXi

Architecture · Hypervisor · Components · Deployment · Post-Install

ESXi Architecture Overview
🔴 Must Know
ESXi is a bare-metal hypervisor built on the VMkernel — a lightweight, purpose-built OS that directly manages all hardware resources, VMs, and critical system processes. It eliminates the traditional "service console."
/* ESXi Two-Tier Architecture */ ┌─────────────────────────────────────────────────────┐ │ USER-WORLD PROCESSES (Top Layer) │ │ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────┐ │ │ │ DCUI │ │ hostd │ │ vpxa │ │ VMs │ │ │ │(Local UI)│ │(Local Mg)│ │(vCtr Agt)│ │ │ │ │ └──────────┘ └──────────┘ └──────────┘ └───────┘ │ ├─────────────────────────────────────────────────────┤ │ VMkernel (Base Layer) │ │ CPU Scheduler │ Memory Mgmt │ Storage │ Network │ │ Resource Scheduling & Isolation │ ├─────────────────────────────────────────────────────┤ │ Physical Hardware (Server) │ │ CPU │ RAM │ Storage │ Network NIC │ └─────────────────────────────────────────────────────┘
💡
Exam Tip: Draw the 2-tier diagram. State that ESXi = bare-metal hypervisor, VMkernel = core OS, no service console. This alone can fetch you 4–5 marks.

Planning a vSphere Deployment
🟠 High Priority

Hardware Requirements

  • CPU: Intel VT-x or AMD-V support (required)
  • Minimum 4 GB RAM (8+ GB recommended)
  • Supported storage controllers & NICs
  • Must be on VMware HCL (Hardware Compatibility List)

Workload Planning

  • Estimate number of VMs needed
  • CPU/RAM/Storage capacity planning
  • Performance requirements (IOPS, throughput)
  • At least 2 hosts recommended for redundancy

Storage Architecture

  • DAS — Direct-Attached Storage
  • NAS — Network-Attached Storage
  • SAN — Storage Area Network
  • vSAN — Hyperconverged (VMware-native)

Network Design

  • Separate VMkernel interfaces for: Management, vMotion, iSCSI, FT
  • Use VLANs for traffic segmentation
  • Multiple NICs for redundancy
💡
Exam Tip: "Planning" questions = mention hardware (HCL, CPU, RAM) + storage options (DAS/NAS/SAN/vSAN) + network design. Cover 3 areas minimum.

vSphere Host Client
🟠 High Priority
vSphere Host Client is a browser-based management interface used to manage a single ESXi host directly. It replaced the old C# thick client.

What you CAN do:

  • Create and manage VMs
  • Manage datastores and networking
  • Set management users and roles
  • Monitor host performance
  • Configure storage, NTP, SSH

Key difference from Web Client:

  • Host Client → manages one ESXi host
  • vSphere Web Client → manages entire vCenter infrastructure
  • Host Client = no vCenter needed
  • Web Client = requires vCenter Server
Network Integration with Infrastructure
🟠 High Priority
Network Integration refers to connecting ESXi hosts into the existing physical network infrastructure using multiple NICs for traffic separation, redundancy, and security.
Traffic TypeVMkernel Port NeededBest Practice
ManagementManagement VMkernelDedicated NIC or VLAN; used for vCenter comm, SSH, DCUI
vMotionvMotion VMkernelDedicated 10 GbE NIC; must not share with management
VM TrafficVM Port GroupSeparate from management; can use VLANs for segmentation
iSCSI / NFSStorage VMkernelDedicated NIC(s) for storage traffic; jumbo frames recommended
Fault ToleranceFT Logging VMkernelDedicated 10 GbE; low-latency path required
  • Use VLANs to segment traffic types on shared physical switches
  • Use multiple NICs (NIC Teaming) for redundancy — if one NIC fails, traffic continues
  • Connect ESXi host NICs to separate physical switches for full redundancy
  • Ensure management NIC is on a dedicated, secured management VLAN
💡
Exam Tip: Network integration = traffic separation (table of 5 types) + NIC redundancy + VLAN segmentation. Draw a simple diagram showing management, vMotion, VM, and storage on separate VMkernel ports.
Time Synchronization (NTP) & Name Resolution (DNS)
🟠 High Priority

Time Synchronization — NTP

  • NTP (Network Time Protocol) keeps all ESXi hosts and VMs in time sync
  • Required for: Kerberos authentication (5-min tolerance), log correlation, certificates
  • Configure in DCUI or Host Client → System → Time Configuration
  • Point to reliable NTP server (e.g., pool.ntp.org or internal NTP)
  • vCenter Server must also sync to the same NTP source

Name Resolution — DNS

  • DNS (Domain Name System) resolves hostnames to IP addresses
  • ESXi must resolve its own hostname (forward + reverse DNS)
  • Required for: vCenter registration, SSO, AD integration, certificates
  • Configure: Primary DNS + Secondary DNS + Search Domain
  • Also configure local /etc/hosts as fallback for critical entries
NTP failure effect: Kerberos auth fails if >5 min drift
DNS failure effect: vCenter cannot communicate with hosts

Chapter 3 — Installing & Configuring vCenter Server

vCenter Server · SSO · Platform Services Controller · Web Client

vCenter Server — Introduction
🔴 Must Know
vCenter Server is a centralized management platform for VMware vSphere that allows administrators to manage multiple ESXi hosts and VMs from a single interface. It is deployed as a VCSA (vCenter Server Appliance) — a pre-configured virtual machine.

Core Functions

  • Centralized management (single-pane-of-glass)
  • Resource pooling and allocation
  • Workflow automation
  • Deep performance monitoring

Advanced Features Enabled

  • vSphere High Availability (HA)
  • Fault Tolerance (FT)
  • Distributed Resource Scheduler (DRS)
  • vMotion and Storage vMotion
Key ComponentRole
vSphere ClientWeb-based UI for managing the entire environment
vCenter DatabaseStores inventory, security roles, performance data
vCenter SSOAuthentication service for the vCenter environment
💡
Exam Tip: "Why is vCenter important?" — Without vCenter, you cannot use HA, DRS, vMotion, or FT. These features ONLY work with vCenter. This is a scoring point.
Deployment: VCSA (Virtual Appliance)
Also available on: Windows Server

vCenter Single Sign-On (SSO)
🔴 Must Know
vCenter SSO is a centralized authentication and identity management service that allows users to log in once and access all vSphere components without re-entering credentials.

Authentication Workflow:

1
User logs in → vSphere Web Client / Host Client
↓
2
Credentials sent to SSO Service
↓
3
SSO verifies via AD / LDAP / vsphere.local
↓
4
If valid → SSO issues a secure token (SAML)
↓
5
Token grants access to all connected VMware services — no further logins

Supported Identity Sources:

Active Directory (AD) LDAP Directories vsphere.local (SSO local)

Key Benefits:

Single Login

One authentication → access all VMware services

Scalability

Grows with infrastructure, no per-host config needed

Improved Security

Token-based, MFA support, reduced password exposure

Central User Management

Add identity sources, assign roles, manage AD groups

💡
Exam Tip: Draw the 5-step SSO flow. Mention: identity sources (AD/LDAP/vsphere.local) + SAML token + benefits. "Explain SSO" = classic 6-mark question.
Platform Services Controller (PSC)
🔴 Must Know
PSC (Platform Services Controller) is the backbone infrastructure component in vSphere 6.x that provides centralized security, identity, and licensing services to vCenter Server. Introduced in vSphere 6.0.
PSC Service / ComponentWhat It Does
vCenter SSOSingle Sign-On — secure authentication for users and services
VMCA (VMware Certificate Authority)Issues and manages SSL certificates; supports custom certs
VECS (VMware Endpoint Cert Store)Stores and manages endpoint certificates
VMware License ServiceCentralized license management; replicated across all PSC instances
vmdir (VMware Directory Service)LDAP directory — stores auth, cert, and license info; multi-master replication
STS (Security Token Service)SAML token exchange for secure inter-service communication
Component ManagerService registration and lookup for VMware products
HTTP Reverse ProxyEntry point for secure communication between services

vSphere Web Client for Administration
🟠 High Priority
vSphere Web Client is the primary browser-based interface that connects to vCenter Server and exposes the full range of administrative features for the entire vSphere environment. Access: https://vcenter-ip/ui

What admins can do:

  • Provision and deploy virtual machines
  • Monitor resource usage and performance (real-time)
  • Manage user access and permissions (RBAC)
  • Configure ESXi hosts, licensing, certificates
  • Control VM lifecycle (power on/off, upgrade hardware version)
  • Access VM consoles through the browser
  • Configure distributed switches, storage, clusters
  • Monitor event logs, alarms, and tasks

Home screen areas:

🖥 Hosts and Clusters 📁 VMs and Templates 💾 Storage (Datastores) 🌐 Networking 📋 Tasks and Events

Key advantages:

  • Cross-platform (any OS, any browser)
  • Third-party plug-in integration
  • Role-based access control (RBAC)
  • AD integration for streamlined auth
💡
Exam Tip: Always contrast Host Client vs Web Client in any answer — Host Client = single ESXi host, Web Client = entire vCenter infrastructure. This shows understanding.
